Overview

The goal of this assignment was to practice the workflow of redesigning a simple website. I had to analyze and identify flaws in an existing interface, create low-fidelity and high-fidelity prototypes for various screen sizes, and build a responsive website based on those prototypes.

Identifying Usability Problems

Picking a Webpage

I chose the homepage of The Glam Kitchen to redesign. I chose this website because I use it often for recipes and I see some areas of improvement in terms of usability.

Finding Problems

In order to effectively redesign the page, I needed to identify its problems first. Each problem can be categorized as a usability problem, learnability problem, or a memorability problem:


Usability:

  • There are links at the bottom of the page to sign up for the blog, follow the creator on instagram, and follow them on pinterest that I believe to be pretty cluttered. Some of the text in this section is highlighted so they look like buttons but they aren't.
  • The nav bar goes off the page and uses horizontal scroll, but there is no indication of that unless you scroll left or right yourself.
  • There are two icons (pinterest and youtube) at the top of the page that don't actually link to anything.
  • The link to the youtube page is in a different location than the other social media links
  • There is no name for the webpage so it's confusing to use when you have multiple pages open

Learnability:

  • The interface is pretty simple to learn because each category of recipes is clearly labeled and interactive components look interactive.
  • There are a lot of items in the nav bar which may confuse new users
  • The website could benefit from more labels for their sections so new users know what they're looking at (most popular/recent recipes, join our blog/follow, etc.)

Memorability:

  • The main components of the website are memorable (where to find recipes)
  • The interaction area (blog signup/ social media links) are confusing and difficult to learn to use as well as memorize.

Accessibility

I used WebAIM WAVE to detect possible accessibility problems. Based on this report, I found that this website has a few problems: mainly missing alt text for images, empty links, and contrast issues. I agree with these problems because images should always have descriptions for those who cannot see them, and links should have some sort of descriptive text that tells you where it's taking you. The contrast issues are also a big concern because it's difficult to use the website if you can't read the text.
